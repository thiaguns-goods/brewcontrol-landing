1. O Que é o BrewControl

O BrewControl é um sistema de gestão (ERP) feito especialmente para cervejarias artesanais brasileiras. Ele cobre todo o ciclo da cerveja — do planejamento do lote até a entrega no cliente — em um único sistema, acessível pelo navegador em qualquer dispositivo.

O sistema é dividido em módulos. Cada usuário vê apenas o que é necessário para a sua função, tornando a operação mais simples e segura.

1.1 Módulos do Sistema

Produção: Planejamento e execução dos lotes de cerveja (brassagem, fermentação, envase)

Almoxarifado: Controle de insumos: malte, lúpulo, levedura, adjuntos e embalagens

Barris / Kegs: Gestão do parque de barris como ativos rastreados, comodatos e devoluções

Comercial: Clientes, pedidos de venda, orçamentos e reservas de produto

Logística: Rotas de entrega, registro de entregas e devoluções

Financeiro: Recebimentos, contas a pagar, fluxo de caixa e relatórios gerenciais

Brewpub / PDV: Atendimento de mesas, cardápio, comanda, caixa e fechamento do turno

PDV Mobile: Controle de torneiras e vendas de chopp no balcão via dispositivo móvel

1.2 Papéis de Usuário

O sistema usa controle de acesso por papel (função). Cada papel libera os módulos e ações correspondentes:

1.3 Glossário Essencial

Brassagem: Processo de produção de um lote de cerveja em um único dia (brew day).

Lote: Conjunto de uma ou mais brassagens da mesma receita.

Mostura: Etapa de conversão do amido do malte em açúcar (mash).

Fermentação: Atuação da levedura convertendo açúcar em álcool e CO₂.

OG / FG: Densidade original (antes) e final (depois) da fermentação.

Barril / Keg: Ativo retornável para armazenar e servir cerveja.

Comodato: Empréstimo de barril ou chopeira para o cliente.

Cigano: Cervejeiro que produz em instalações de terceiros (lote terceirizado).

P1 / P2: P1 = Planejamento do lote; P2 = Execução (Brew Day).

CMV: Custo de Mercadoria Vendida — custo direto dos insumos de um produto.

KDS: Kitchen Display System — tela da cozinha que exibe pedidos em tempo real.

Comanda: Registro de consumo de uma mesa, encerrada no pagamento.

Tenant: Conta de uma cervejaria no sistema (multi-tenant = múltiplos clientes).

2. Primeiro Acesso e Navegação

2.1 Entrando no Sistema

  1. Acesse app.brewcontrol.app.br no navegador (Chrome ou Edge recomendados).
  2. Digite o e-mail e a senha fornecidos pelo administrador da sua cervejaria.
  3. Na primeira vez, o sistema pode pedir para você trocar a senha.
  4. Após o login, você verá o painel principal com os módulos liberados para o seu papel.

DICA O BrewControl funciona no computador, tablet e celular. Em celulares, use o menu hambúrguer (☰) para navegar entre os módulos.

ATENÇÃO Se você vir 'Acesso negado' em algum módulo, peça ao administrador para verificar o seu papel de usuário.

2.2 Navegação Principal

A barra lateral esquerda é o menu principal. Ela mostra apenas os módulos liberados para o seu papel. Clique em qualquer item para acessar o módulo.

No topo da tela você encontra:

  • Nome da cervejaria ativa (se você pertence a mais de uma)
  • Sino de notificações: alertas de estoque baixo, barris pendentes, entregas atrasadas
  • Avatar com seu nome: acesso às configurações da sua conta e botão de sair

2.3 Padrões Visuais do Sistema

O BrewControl usa cores e ícones padronizados para facilitar a leitura rápida:

  • Verde = ativo, concluído, aprovado
  • Amarelo = em andamento, aguardando, atenção
  • Vermelho = atrasado, crítico, erro, cancelado
  • Azul = informativo, planejado, reservado
  • Badges numerados = quantidade de itens pendentes naquele contexto

3. Configuração Inicial — Administrador

Esta seção é para o administrador que está configurando o sistema pela primeira vez. Siga o checklist abaixo antes de liberar o acesso para a equipe.

3.1 Dados da Cervejaria

Acesse Admin → Configurações para preencher:

  • Nome fantasia e razão social
  • CNPJ e endereço completo
  • Logotipo (usado em documentos PDF e recibos)
  • Configurações de notificação (e-mail para alertas críticos)

3.2 Criando Usuários

  1. Acesse Admin → Usuários → Novo Usuário.
  2. Preencha nome, e-mail e defina o papel (função).
  3. O sistema enviará um e-mail de convite com o link de primeiro acesso.
  4. O usuário define a própria senha no primeiro login.

BOA PRÁTICA Crie um usuário com papel Viewer para o sócio ou contador. Assim eles podem acompanhar os números sem risco de alterar dados.

3.3 Configurando Equipamentos de Produção

Em Admin → Equipamentos, cadastre os equipamentos da sua fábrica:

  • Tipo: monobloco, bibloco, tribloco, pentabloco
  • Capacidade de brassagem (litros)
  • Nome identificador (ex: 'Tribloco 400L')

Os equipamentos são selecionados durante o planejamento dos lotes, então é importante cadastrá-los antes de começar a usar o módulo de Produção.

3.4 Configurando Tanques

Em Admin → Tanques, cadastre seus tanques de fermentação/maturação:

  • Nome do tanque (ex: 'Fermentador 1', 'Tank 500L')
  • Capacidade em litros
  • Tipo: fermentador, maturador, bright beer tank, unitank

O sistema usa os tanques para calcular disponibilidade e mostrar o que está ocupado no painel de produção.

3.5 Perfis de Produção

Antes de cadastrar receitas, crie os perfis que serão reutilizados:

  • Perfis de Mostura: temperaturas e tempos de cada etapa (ex: 'Pale Ale Básico: 67°C por 60min')
  • Perfis de Fermentação: temperatura, tempo, diacetil rest (ex: 'Lager Fria: 10°C por 14 dias')
  • Perfis de Água: ajuste de minerais por estilo (ex: 'Água Pilsen: dureza 50 ppm')

Acesse: Produção → Perfis → [Mostura / Fermentação / Água]

4. Módulo de Produção

O módulo de Produção é dividido em duas fases: P1 (Planejamento) e P2 (Execução — Brew Day). Um lote de cerveja começa sempre no P1 e avança para o P2 quando a produção começa.

4.1 Fase P1 — Planejamento do Lote

4.1.1 Criando uma Receita

Acesse Produção → Receitas → Nova Receita.

  1. Defina o nome da cerveja, estilo (Ex: American IPA, Pilsen Alemã) e cor EBC.
  2. Defina o volume de brassagem (litros que sairão do equipamento).
  3. Adicione os ingredientes: malte base, maltes especiais, lúpulos, levedura, adjuntos e água.
  4. Para cada ingrediente: nome, quantidade (kg/g/ml/un), ponto de adição (mostura, fervura, dry hop).
  5. Associe um perfil de mostura e um perfil de fermentação.
  6. Salve a receita. Ela ficará disponível para criar lotes.

DICA Use o campo 'Notas da Receita' para registrar observações do cervejeiro (ex: 'dry hop por 5 dias a 18°C para aroma mais intenso').

4.1.2 Criando um Lote de Produção

Acesse Produção → Lotes → Novo Lote.

  1. Selecione a receita.
  2. Defina o volume total do lote (pode ser maior que uma brassagem).
  3. O sistema calcula automaticamente quantas brassagens são necessárias.
  4. Escolha o tanque de destino.
  5. Defina a data prevista de início da brassagem.
  6. Salve. O lote aparecerá no painel de produção com status 'Planejado'.

4.1.3 Planejando as Brassagens

Cada lote pode ter uma ou mais brassagens (quando o volume é maior que a capacidade do equipamento). Para cada brassagem:

  • Selecione o equipamento de brassagem
  • Defina a data prevista
  • O sistema mostra a disponibilidade do equipamento naquela data
  • Confirme para criar a brassagem em status 'Agendada'

4.2 Fase P2 — Brew Day (Execução)

Acesse Produção → Brassagens → [selecione a brassagem do dia] → Iniciar Brew Day.

O Brew Day é guiado por um wizard (assistente passo a passo). Cada etapa deve ser concluída antes de avançar para a próxima.

→ Mostura: Registre temperatura de entrada da água, temperatura atingida no mash e tempo de repouso.

→ Filtragem: Registre o volume de mosto coletado e a densidade pré-fervura (OG pré-fervura).

→ Fervura: Registre as adições de lúpulo com horário. O sistema usa o perfil da receita como guia.

→ Resfriamento: Registre a temperatura de saída e o volume transferido para o fermentador.

→ Transferência: Selecione o tanque de destino e confirme o volume transferido.

→ OG: Registre a densidade original (OG) medida no densímetro ou refratômetro.

→ Inoculação: Registre a levedura utilizada, quantidade e temperatura de inoculação.

→ Fermentação: Registre leituras periódicas de temperatura e densidade (FG). O sistema rastreia o progresso.

→ Condicionamento: Registre ajustes finais: dry hop, correções de sabor, carbonatação.

→ Envase: Registre o volume envasado em cada barril ou embalagem.

IMPORTANTE Os eventos de produção são imutáveis: nunca são apagados. Se errou algum dado, registre um novo evento de correção com o motivo. Isso garante rastreabilidade total do lote.

4.3 Lote Terceirizado (Cigano)

Se você produz em instalações de terceiros ou presta serviço de produção para outro cervejeiro:

  1. Na criação do lote, ative a opção 'Lote Terceirizado'.
  2. Informe o nome da cervejaria cliente.
  3. O sistema marca o lote com origem 'terceirizado' e pula o consumo automático de estoque (pois os insumos são do cliente, não do seu almoxarifado).

5. Módulo de Almoxarifado

O Almoxarifado controla todos os insumos da fábrica: maltes, lúpulos, leveduras, adjuntos, embalagens e materiais auxiliares. O saldo de estoque é calculado automaticamente com base nas entradas e saídas registradas.

5.1 Cadastrando Insumos

Acesse Almoxarifado → Insumos → Novo Insumo.

  • Nome e categoria (malte, lúpulo, levedura, adjunto, embalagem, auxiliar)
  • Unidade de medida (kg, g, L, ml, unidade, caixa)
  • Estoque mínimo de alerta (o sistema avisa quando ficar abaixo desse nível)
  • Fornecedor preferencial (quando o módulo de Fornecedores estiver ativo)

5.2 Registrando Entradas

  1. Acesse Almoxarifado → Movimentações → Nova Entrada.
  2. Selecione o insumo e o fornecedor.
  3. Informe a quantidade recebida, o valor total da nota e a data.
  4. O sistema atualiza o saldo automaticamente.

BOA PRÁTICA Registre as entradas no mesmo dia do recebimento. Isso mantém o saldo sempre preciso e evita surpresas na hora de planejar uma brassagem.

5.3 Consumo Automático vs. Manual

O consumo de insumos acontece de duas formas:

  • Automático: ao registrar eventos de produção no Brew Day, o sistema debita automaticamente os insumos da receita do estoque.
  • Manual: acesse Movimentações → Nova Saída para registrar perdas, amostras para análise ou transferências com motivo obrigatório.

5.4 Inventário

  1. Acesse Almoxarifado → Inventário → Iniciar Contagem.
  2. O sistema mostra o saldo teórico (calculado) de cada insumo.
  3. Digite a quantidade física contada.
  4. O sistema gera automaticamente os ajustes de entrada ou saída com log auditável.
  5. Confirme o inventário. As divergências ficam registradas no histórico.

6. Módulo de Barris e Ativos

Cada barril é um ativo rastreado individualmente. O sistema usa eventos para registrar tudo que acontece com o barril: enchimento, entrega, devolução, manutenção. O status do barril é sempre calculado com base no histórico de eventos — nunca editado diretamente.

6.1 Cadastrando Barris

Acesse Ativos → Barris → Novo Barril.

  • Código ou número do barril (pode ser o código gravado no barril físico)
  • Capacidade (30L, 50L, etc.)
  • Tipo (inox, PET, growler)
  • Data de aquisição

Após o cadastro, o barril entra com status 'Disponível'.

6.2 Fluxo do Barril

Disponível: Barril limpo, vazio e pronto para encher.

Cheio: Envasado com produto. Associado a um lote de produção.

Reservado: Separado para um pedido. Ainda não saiu da fábrica.

Em Campo: Entregue ao cliente em comodato.

Aguard. Dev.: Cliente confirmou devolução, aguardando recolhimento.

Em Manutenção: Barril com defeito em reparo.

6.3 Enchendo um Barril

  1. Após o envase no Brew Day, o sistema registra automaticamente o barril como 'Cheio'.
  2. Ou acesse Ativos → Barris → [barril] → Registrar Envase.
  3. Informe o lote de origem e o volume envasado.
  4. O status muda para 'Cheio' e o barril fica disponível para pedidos.

6.4 Comodato — Barril em Campo

Quando um barril é entregue ao cliente em comodato, o sistema rastreia automaticamente:

  • Data da saída e cliente responsável
  • Tempo em campo (dias)
  • Alerta de comodato vencido (configurável: ex. 30 dias sem devolução)

A devolução é registrada no módulo de Logística no momento da recolha.

7. Módulo Comercial

O módulo Comercial gerencia os clientes B2B, pedidos de venda e reservas de produto. Um pedido criado aqui reserva produto logicamente — o estoque só baixa de fato quando a entrega é executada no módulo de Logística.

7.1 Cadastrando Clientes

  1. Acesse Comercial → Clientes → Novo Cliente.
  2. Preencha: razão social, nome fantasia, CNPJ, e-mail, telefone, endereço de entrega.
  3. Defina a condição de pagamento padrão (ex: Boleto 30 dias, À vista no ato).
  4. Opcionalmente, cadastre quantos barris esse cliente tem em comodato como limite máximo.
  5. Salve. O cliente já aparece disponível na criação de pedidos.

7.2 Criando um Pedido de Venda

  1. Acesse Comercial → Pedidos → Novo Pedido.
  2. Selecione o cliente.
  3. Defina a data de entrega desejada.
  4. Adicione os itens: produto (cerveja, barril, embalagem), quantidade, preço.
  5. Selecione a condição de pagamento para este pedido.
  6. Se o pedido inclui chopeira ou cilindro de CO₂, selecione o ativo específico para reservar.
  7. Confirme o pedido. O status vai para 'Confirmado' e o produto é reservado.

DICA O campo 'Observações da entrega' aparece para o motorista no momento da entrega. Use para informar horário de recebimento, portaria ou pessoa de contato.

7.3 Acompanhando Pedidos

Na lista de pedidos, você vê todos os pedidos com seus status:

  • Rascunho: em edição, ainda não confirmado
  • Confirmado: reserva feita, aguardando entrega
  • Em rota: entregador saiu com o pedido
  • Entregue: entrega confirmada, estoque baixado
  • Cancelado: pedido cancelado com motivo registrado

8. Módulo de Logística

O módulo de Logística organiza as entregas por rotas. É aqui que o estoque é efetivamente baixado e os ativos (barris, chopeiras, cilindros) são transferidos para o cliente.

8.1 Planejando uma Rota de Entrega

  1. Acesse Logística → Rotas → Nova Rota.
  2. Defina a data e o veículo/motorista responsável.
  3. Adicione os pedidos confirmados que serão entregues nessa rota.
  4. O sistema calcula a distância estimada com base nos CEPs dos clientes.
  5. Organize a sequência de paradas (arraste e solte na ordem desejada).
  6. Salve a rota. Ela fica disponível para execução no dia da entrega.

8.2 Executando a Entrega

  1. No dia da entrega, acesse a rota e clique 'Iniciar Rota'.
  2. Para cada parada: confirme os itens entregues, registre quem recebeu.
  3. Se houver devolução de barril, registre os barris devolvidos nesta parada.
  4. Ao concluir a parada, o status do pedido muda para 'Entregue' e o estoque é baixado.
  5. Ao concluir a última parada, finalize a rota.

ATENÇÃO Somente a execução da entrega baixa o estoque. Criar o pedido no Comercial apenas reserva o produto — não desconta do saldo.

9. Módulo Financeiro

O Financeiro do BrewControl é gerencial — mostra o fluxo de caixa real da cervejaria, contas a receber, contas a pagar e indicadores de desempenho. Não substitui o sistema contábil fiscal, mas é o painel de controle financeiro do dia a dia.

9.1 Lançamentos Financeiros

Todo movimento de dinheiro é um lançamento. Lançamentos são criados automaticamente pelo sistema (ex: quando uma entrega é concluída, gera uma conta a receber) ou manualmente.

Tipos de lançamento:

  • Receita: dinheiro que entra (pagamento de pedido, venda no brewpub)
  • Despesa: dinheiro que sai (compra de insumo, conta fixa, salário)
  • Transferência: movimento entre contas (ex: caixa físico → conta bancária)

9.2 Registrando um Recebimento

  1. Acesse Financeiro → Contas a Receber.
  2. Localize o pedido ou lançamento pendente.
  3. Clique em 'Registrar Recebimento'.
  4. Informe a data, o valor recebido e a forma de pagamento (PIX, boleto, cartão, dinheiro, transferência).
  5. Confirme. O lançamento passa para status 'Recebido'.

9.3 Dashboard Financeiro

O painel financeiro mostra:

  • Total a receber (no mês atual e próximos 30 dias)
  • Total recebido no mês
  • Total de despesas no mês
  • Saldo do período
  • Gráfico de entradas vs. saídas por semana
  • Lista de parcelas em atraso com destaque em vermelho

9.4 Planos de Assinatura

Acesse Financeiro → Plano para visualizar e gerenciar sua assinatura do BrewControl. Você vê o plano atual, a data de vencimento e os módulos incluídos. Módulos não incluídos no plano aparecem travados com botão 'Fazer upgrade'.

10. Módulo Brewpub — Atendimento e PDV

O módulo Brewpub é o sistema de ponto de venda do bar. Funciona para cervejarias com taproom, brewpub completo com cozinha ou bar de chope. O sistema adapta automaticamente o que aparece conforme os módulos ativos da sua cervejaria.

10.1 Perfis de Operação

10.2 Mapa de Mesas

A tela principal do Brewpub é o mapa de mesas. Cada mesa aparece como um card colorido:

  • Verde: mesa livre
  • Vermelho: mesa ocupada — mostra garçom, valor acumulado e tempo de ocupação
  • Amarelo: aguardando atendimento
  • Azul: mesa reservada

Para abrir uma mesa:

  1. Toque no card da mesa desejada.
  2. Informe o número de pessoas e o nome ou PIN do garçom responsável.
  3. A mesa abre e aparece como 'Ocupada' para todos os dispositivos.

10.3 Lançando Itens na Comanda

  1. Toque na mesa ocupada → Lançar pedido.
  2. Navegue pelas categorias do cardápio (Chopes, Drinks, Petiscos, Pratos...).
  3. Toque no item e ajuste a quantidade.
  4. Adicione observações se necessário (ex: 'sem gelo', 'bem passado').
  5. Confirme. O pedido vai automaticamente para a impressora correta:
  • Bebidas → impressora do bar
  • Comida → impressora da cozinha / KDS
  1. O garçom recebe confirmação na tela.

DICA Itens podem ser adicionados em várias rodadas sem fechar a comanda. A comanda só fecha quando o cliente paga.

10.4 Cancelando um Item

  1. Acesse a comanda → toque no item → Cancelar item.
  2. Digite o motivo do cancelamento.
  3. Se necessário, informe o PIN do gerente para autorizar.
  4. O cancelamento fica registrado no log auditável.

10.5 Transferências e Divisões

Situações especiais que o sistema suporta:

  • Transferir item para outra mesa: útil quando o cliente muda de lugar
  • Transferir mesa inteira: mesa 4 vira mesa 7, por exemplo
  • Dividir conta: em partes iguais ou por seleção manual de itens por pessoa
  • Juntar mesas: agrupa duas mesas em uma comanda única

10.6 Cardápio e Torneiras

O cardápio é configurado em Admin → Brewpub → Cardápio. Cada item tem:

  • Nome, descrição, foto e preço
  • Categoria (Chopes, Cervejas, Drinks, Petiscos, Pratos, Sobremesas)
  • Destino de impressão: bar ou cozinha
  • Status ativo/inativo (para tirar do cardápio temporariamente)

Torneiras e Chope:

  • Cada torneira é vinculada a um barril físico do módulo Ativos
  • Ao lançar um copo de chope, o volume é debitado do barril automaticamente
  • O sistema mostra o nível estimado de cada torneira em tempo real
  • Alerta automático quando o barril está próximo de acabar
  • Quando o barril acaba, a torneira desaparece do cardápio do garçom automaticamente

10.7 Fechamento de Mesa e Pagamento

  1. Acesse a mesa → Fechar Comanda.
  2. Revise os itens e o valor total.
  3. Selecione a forma de pagamento: dinheiro, PIX (QR code gerado na tela), cartão crédito/débito, voucher ou misto.
  4. Para divisão de conta: selecione 'Dividir' antes do pagamento.
  5. Confirme o pagamento.
  6. O sistema registra a venda no financeiro automaticamente.
  7. O cupom de consumo (não fiscal) pode ser impresso ou enviado.

10.8 Caixa e Turno

Abertura do turno:

  1. Acesse Brewpub → Caixa → Abrir Turno.
  2. Informe o operador responsável e o saldo inicial em dinheiro.

Durante o turno:

  • Sangria: retirada de dinheiro com motivo registrado
  • Suporte: múltiplos operadores podem trabalhar no mesmo turno

Fechamento do turno:

  1. Acesse Brewpub → Caixa → Fechar Turno.
  2. Informe o valor contado em caixa por forma de pagamento.
  3. O sistema compara com o valor calculado e mostra a diferença.
  4. Confirme o fechamento. O relatório do turno fica disponível para impressão.

11. PDV Mobile — Balcão e Torneiras

O PDV Mobile é uma tela pública acessada pelo endereço brewcontrol.app.br/brewpub — sem necessidade de login, protegida por PIN. É usada no tablet do balcão para controle de chopp nas torneiras.

11.1 Acessando o PDV Mobile

  1. Abra o link brewcontrol.app.br/brewpub no tablet ou computador do balcão.
  2. Digite o PIN de 4 dígitos configurado pelo administrador.
  3. A tela de torneiras aparece com o status de cada barril em tempo real.

11.2 Registrando uma Venda de Chopp

  1. Toque na torneira correspondente.
  2. Toque em '+1 Copo' para registrar uma venda.
  3. O sistema debita o volume do barril e registra a venda no histórico.
  4. Para registrar uma perda (espuma excessiva, derramamento): toque em 'Registrar Perda'.

11.3 Modo Simples vs. Modo Copos

  • Modo Simples: apenas registra +1 venda por toque (sem escolher o tamanho do copo)
  • Modo Copos: o operador escolhe o tamanho do copo a cada venda (300ml, 400ml, 500ml)

O modo é configurado pelo administrador em Admin → Brewpub → Configurações do PDV.

11.4 Encerramento de Sessão

  1. Acesse o menu → Encerrar Sessão.
  2. O sistema exibe um resumo: vendas do turno por torneira, volume total vendido, total de perdas.
  3. Confirme para encerrar. Os dados ficam no histórico.

13. Rotinas Diárias por Função

13.1 Checklist de Abertura — Caixa / Gerente

  1. Acesse o Brewpub → verificar mesas abertas (se alguma ficou do dia anterior).
  2. Abra o turno de caixa com o saldo inicial.
  3. Verifique o nível de cada torneira no painel de Ativos → Barris.
  4. Cheque o painel de Almoxarifado por alertas de estoque mínimo.
  5. Confirme as entregas do dia em Logística → Rotas.

13.2 Rotina de Produção — Cervejeiro

  1. Verifique a brassagem do dia em Produção → Brassagens.
  2. Registre as leituras de fermentação dos tanques ativos (temperatura, densidade).
  3. Execute os eventos do Brew Day se houver brassagem programada.
  4. Registre qualquer ajuste (adição de lúpulo, nutrientes, corrações de pH) como evento.

13.3 Rotina Comercial

  1. Verifique os pedidos com entrega prevista para hoje em Comercial → Pedidos.
  2. Confirme com a logística que os pedidos estão em rota.
  3. Registre novos pedidos recebidos por WhatsApp, e-mail ou visita.
  4. Acompanhe os recebimentos pendentes em Financeiro → Contas a Receber.

13.4 Checklist de Fechamento — Caixa / Gerente

  1. Feche o turno de caixa: conte o dinheiro e confirme o fechamento.
  2. Verifique se todas as mesas foram fechadas.
  3. Confirme as entregas do dia como 'Entregues' no módulo Logística.
  4. Revise os alertas do dia no sino de notificações.

14. Perguntas Frequentes e Erros Comuns

P: Não consigo ver um módulo no menu.

R: Seu usuário provavelmente não tem o papel necessário. Peça ao administrador para verificar suas permissões em Admin → Usuários.

P: O saldo de estoque está errado.

R: Acesse o histórico de movimentações do insumo (Almoxarifado → Insumos → [item] → Histórico). Verifique se houve consumo automático não esperado após um evento de produção. Para correção: faça um inventário e registre o ajuste com motivo.

P: O barril sumiu do estoque.

R: O barril provavelmente foi entregue e está 'Em Campo'. Acesse Ativos → Barris e filtre por status 'Em Campo'.

P: O pedido não está avançando para 'Entregue'.

R: A entrega precisa ser registrada no módulo Logística → Rotas. Só o registro da execução da entrega muda o status do pedido.

P: A comanda sumiu no Brewpub.

R: Verifique se a mesa foi fechada por outro operador. Acesse Brewpub → Histórico de Comandas para localizar.

P: O financeiro não está mostrando um pagamento que fizemos.

R: O pagamento precisa ser registrado manualmente em Financeiro → Contas a Receber → Registrar Recebimento. Entregas confirmadas geram a conta a receber, mas o recebimento em si sempre precisa de confirmação.

P: Errei um dado em um evento de produção.

R: Eventos de produção não podem ser editados ou apagados. Registre um novo evento de correção com o valor correto e o motivo da correção. O sistema mantém ambos no histórico para rastreabilidade.

P: O nome do meu plano está como 'Trial'.

R: O período de avaliação gratuita está ativo. Acesse Financeiro → Plano para contratar um plano ou entre em contato pelo suporte.

15. Boas Práticas de Uso

15.1 Para a Equipe de Produção

  • Registre os eventos no mesmo dia em que acontecem. Dados retroativos reduzem a precisão dos relatórios.
  • Sempre leia a densidade no início e no fim da fermentação — são dados críticos para o cálculo do teor alcoólico.
  • Use o campo de Notas para registrar observações que não cabem nos campos estruturados.
  • Quando um lote apresentar desvio (temperatura fora do perfil, OG errada), registre o evento de desvio imediatamente com o motivo.

15.2 Para o Comercial

  • Preencha o endereço completo dos clientes, incluindo CEP. O sistema usa o CEP para calcular distância e sugerir frete.
  • Defina a condição de pagamento padrão no cadastro do cliente para agilizar a criação de pedidos.
  • Use o campo de Observações para registrar combinados especiais (ex: 'prefere entrega às terças pela manhã').

15.3 Para o Financeiro

  • Registre os recebimentos no mesmo dia do depósito ou transferência. Isso mantém o fluxo de caixa preciso.
  • Categorize cada lançamento corretamente — isso alimentará futuramente o DRE e o fluxo por competência.

15.4 Para o Brewpub

  • Sempre abra o turno antes de começar a atender. O fechamento de caixa depende da abertura correta.
  • Treine garçons para lançar os itens no momento do pedido, não no fechamento da mesa.
  • Configure o estoque mínimo dos barris para receber alertas antes de acabar, evitando torneiras secas.
